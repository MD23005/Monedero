# Sistema de Control de Gastos Personales

## Descripción del Proyecto

El presente proyecto consiste en el desarrollo de una aplicación web orientada al control de gastos personales. El sistema permite a los usuarios registrar, consultar, actualizar y eliminar sus gastos, así como visualizar información relevante sobre su situación financiera mediante un dashboard.

El propósito principal es brindar una herramienta sencilla que facilite la organización de los gastos y contribuya a una mejor toma de decisiones económicas.

---

## Objetivo General

Desarrollar una aplicación web que permita gestionar los gastos personales mediante la implementación de funcionalidades CRUD, almacenamiento local, consumo de APIs modernas y visualización de datos en un dashboard.

---

## Objetivos Específicos

* Implementar operaciones CRUD (crear, leer, actualizar y eliminar) para la gestión de gastos.
* Aplicar manejo de eventos, funciones y validaciones en JavaScript.
* Manipular el DOM para la visualización dinámica de la información.
* Implementar manejo de errores utilizando estructuras try/catch.
* Utilizar LocalStorage como mecanismo principal de almacenamiento.
* Implementar SessionStorage o cookies como almacenamiento complementario.
* Consumir una API REST moderna utilizando Fetch.
* Utilizar geolocalización para enriquecer la funcionalidad del sistema.
* Mostrar métricas en un dashboard (totales, categorías, etc.).
* Implementar Web Workers para el procesamiento de datos sin afectar el rendimiento de la interfaz.

---

## Tecnologías Utilizadas

* HTML5
* CSS3
* JavaScript
* Fetch API
* LocalStorage
* SessionStorage
* Web Workers

---

## Segmento Objetivo

El sistema está dirigido a:

* Estudiantes que desean administrar sus gastos personales.
* Personas que buscan llevar un control básico de sus finanzas.
* Usuarios con ingresos variables que necesitan organizar sus egresos.
* Cualquier persona interesada en mejorar su control financiero diario.

---

## Funcionalidades del Sistema

* Registro de gastos por categorías (alimentación, educación, transporte, entre otros).
* Edición de gastos registrados.
* Eliminación de registros.
* Visualización de lista de gastos.
* Almacenamiento local de la información en el navegador.
* Dashboard con métricas generales de gastos.
* Consumo de API externa (por ejemplo, geolocalización).
* Procesamiento de datos mediante Web Workers.

---

## Estructura del Proyecto

El proyecto está organizado en componentes frontend que interactúan con el almacenamiento local y servicios externos mediante Fetch. Adicionalmente, se puede utilizar una API desarrollada en Express para pruebas o ampliaciones del sistema.
